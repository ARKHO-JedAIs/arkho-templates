## Configuration

All configuration is read from build-time environment variables (see
`.env.example`). The app validates them at startup with Zod and shows an explicit
configuration-error screen if any required value is missing or invalid. Changing
an environment value requires a rebuild.

| Variable | Required | Purpose |
|---|---|---|
| `VITE_AWS_REGION` | yes | Cognito user pool region. |
| `VITE_COGNITO_USER_POOL_ID` | yes | Cognito user pool id. |
| `VITE_COGNITO_USER_POOL_CLIENT_ID` | yes | Cognito app client id (public). |
| `VITE_COGNITO_DOMAIN` | no | Reserved for future hosted/federation flows. |
| `VITE_API_BASE_URL` | yes | Backend API base URL. |

Never commit a real `.env`; only `.env.example` is tracked.

## Project structure (atomic design)

```
src/
├── app/          # router, layout shell, auth guard
├── components/
│   ├── atoms/      # shadcn/ui primitives (button, input, card, label)
│   ├── molecules/  # small composites (form field)
│   ├── organisms/  # larger composites (site header)
│   └── templates/  # page-level layouts and the config-error screen
├── pages/        # route screens
├── features/     # per-domain hooks, schemas, and stores
├── lib/          # config (env + Zod), http (axios), auth (Amplify)
└── test/         # Vitest setup and example tests
```

## Template version

This project records the producing template name and version in `arkho.json`,
written by `arkho-cli` at generation time (`template.name`, `template.version`,
`template.tag`, `template.commit`). Template versions are published as git tags
in the namespace `react-spa/v<semver>`; the CLI uses the recorded version to
propose upgrades.

## Project lineage

This project was generated by `arkho-cli`. The original template lives in the
